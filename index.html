Filtrer la base SIRENE

12 millions d'entreprises, c'est trop. Il faut filtrer.

Par secteur

Code NAF = code activité

Ex: 6201Z = Programmation informatique
Ex: 7022Z = Conseil en gestion

Par zone

Code postal, département, région

Ex: 75* = Paris
Ex: 69* = Rhône

Par taille

Effectif salarié

Ex: 10-49 = PME
Ex: 50-249 = ETI

Par ancienneté

Date de création

Ex: > 2020 = startups récentes
Ex: < 2000 = entreprises établies

13

Les limites du scraping

À savoir

Google n'aime pas qu'on scrape. Des limites existent :

  • Blocage d'IP si trop de requêtes
  • Captchas
  • Conditions d'utilisation
  • RGPD à respecter

Solutions : proxies, rotation d'IP, respect des rate limits, utiliser les API officielles quand elles existent.

n8n - Comment ça marche

Tu crées des "workflows" visuels. Chaque bloc est une action.

Trigger

Ce qui déclenche le workflow. Ex: nouveau fichier, heure fixe, webhook.

Nodes

Les actions à exécuter. Ex: lire un CSV, appeler une API, filtrer des données.

Output

Le résultat final. Ex: sauvegarder dans Google Sheets, envoyer un email.

Des données sales = des campagnes ratées.

  • Emails valides (format + existence)
  • Doublons
  • Entreprises fermées
  • Infos obsolètes
  • Données manquantes

Outils

  • NeverBounce (validation email)
  • ZeroBounce
  • Hunter.io (vérification)
  • OpenRefine (nettoyage)
29

La déduplication

Même personne dans plusieurs sources = doublon à éliminer.

Critères de matching :
├── Email identique → Doublon certain
├── Nom + Entreprise identiques → Probablement doublon
├── Téléphone identique → Vérifier
└── SIRET identique → Même entreprise

Action : garder l'entrée la plus complète

Éviter les spams

Un email en spam = un email jamais lu. Voici les règles.

Technique

  • SPF configuré
  • DKIM configuré
  • DMARC configuré
  • Domaine dédié à la prospection

Réputation

  • Warm-up du domaine
  • Volume progressif
  • Taux de bounce bas
  • Peu de signalements spam

Contenu

  • Pas de mots spam
  • Ratio texte/image correct
  • Lien de désinscription
  • Personnalisation
35

Le warm-up

Un nouveau domaine qui envoie 10K emails d'un coup = spam direct.

Il faut "chauffer" le domaine progressivement.

Planning type de warm-up :

Semaine 1 : 20 emails/jour
Semaine 2 : 50 emails/jour
Semaine 3 : 100 emails/jour
Semaine 4 : 200 emails/jour
Semaine 5 : 400 emails/jour
...

Durée recommandée : 4-8 semaines

Outils de warm-up : Lemwarm, Mailreach, Warmbox
